95. **DOUBLE-HASH D'AUTH ENTRE CLIENT ET WORKER = 401 silencieux sur 100% des appels, masqué par un `/health` sans auth (Apex v13.4.322, Kevin 2026-06-08 « code pris mais IA KO »)** — Après avoir débloqué le stockage du code admin (`ax_pin_kdmc_admin`), l'IA restait KO. Le **worker** (`verifyPin`) re-hashait ce header (`sha256(header)`) puis comparait à `env.APEX_ADMIN_PIN_SHA256` (= `sha256(PIN)`) → il comparait `sha256(sha256(PIN))` à `sha256(PIN)` → **jamais égal → 401 sur chaque provider**.
