## Bugs CRITIQUES (corrigés)

### BUG-1 — IA crash silencieux quand user demande un studio
- **Sévérité** : CRITIQUE
- **Lieu** : `_buildSystemPrompt()` ligne 3661, `sendMessage()` ligne 3633, `vMain()` switch ligne 3555
- **Symptôme** : Le system prompt v12.176/v12.213 dit à l'IA "appelle `axOpenStudio('video')` quand user demande un montage". `sendMessage` appelle `axDetectIntent(text)` puis `axProposeStudio(...)`. `vMain()` route 10 vues `studiovideo`/`studiomusic`/etc. **AUCUNE de ces fonctions n'était définie**. Quand Kevin disait "fais-moi une video" → `axDetectIntent is not a function` → catch silencieux → IA répond mal.
- **Impact** : feature majeure (Smart Studios v12.176) totalement cassée pour 12+ studios.
- **Fix appliqué** (v12.220 ligne 1108-1115) : ajout de `axDetectIntent`, `axOpenStudio`, `axProposeStudio`, `axGetCurrentProfileSummary` (référencé en ligne 7900+) + auto-stub IIFE pour les 15 vues `vStudio*`/`vPlantStudio`/`vGeoStudio`/etc. manquantes (renvoient un message + bouton "Aller au chat").
- **Validation** : `node --check` OK, vue dégradée propre au lieu d'un crash.

## Bugs HAUTS (corrigés)

### BUG-2 — `_setApiKey` sans guard admin + bypass sync Firebase
- **Sévérité** : HAUTE
- **Lieu** : ligne 3660 `function _setApiKey(k){if(k)localStorage.setItem("ax_api_key",k);else localStorage.removeItem("ax_api_key");}`
- **Symptôme** : Tout user (même non-admin) peut appeler `_setApiKey(...)` (exposé via `window`). De plus, écriture directe `localStorage.setItem` au lieu de `ls()` → la clé n'est PAS synchronisée vers Firebase pour les autres appareils Kevin (cf. erreur connue #27 KDMC v12.2).
- **Impact** : sécurité (exposition possible) + UX (clé non sync entre iPhone/iPad/PC).
- **Fix appliqué** (v12.220) : guard `if(!axIsAdmin())return;` + utilisation de `ls()` qui passe par `fbShouldSync` → sync auto vers Firebase + auto-set de `ax_shared_api_key` pour cohérence cross-device.

### BUG-3 — Vues Studio routées vers fonctions inexistantes
- **Sévérité** : HAUTE
- **Lieu** : `vMain()` ligne 3555, switch case "studiovideo", "studiomusic", "studioprefecture", "studiocv", "studiofacture", "studiocontrat", "studiopresentation", "studioclip", "studiologo", "studios", "plantstudio", "geostudio", "buildingstudio", "gardenlunarstudio", "petstudio".
- **Symptôme** : 15 cases dans le switch appellent des fonctions `vStudioVideo()`/etc. qui n'existent pas → ReferenceError → render() crashe → app affiche le fallback d'erreur.
- **Impact** : tous les liens "Studio" crashaient l'app.
- **Fix appliqué** (v12.220) : IIFE qui crée des stubs `window[name] = function(){return '<div>...</div>'}` pour chacun, avec bouton retour chat.

## Bugs MOYENS (à voir)

### BUG-4 — `_chatPageSize` shadowing dans `vChat()`
- **Sévérité** : MOYENNE
- **Lieu** : ligne 3626 `var _chatPageSize=200;` à l'intérieur de `vChat()` — alors que la globale ligne 3464 vaut 50.
- **Impact** : le bouton "Charger 50 messages plus anciens" modifie la globale (=> 100, 150...) mais `vChat` lit toujours sa locale (200). Le compteur n'avance pas. UX cassée pour la pagination chat.
- **Fix proposé** : retirer `var _chatPageSize=200;` ligne 3626 OU renommer la globale.

### BUG-5 — `axRunCode` sans guard admin
- **Sévérité** : MOYENNE (sandbox iframe limite l'impact)
- **Lieu** : ligne 4091 `function axRunCode(code){...iframe.contentWindow.eval(code)...}`
- **Impact** : Tout user peut exécuter du code arbitraire. La sandbox `allow-scripts` empêche l'accès au DOM parent, mais permet quand même fetch réseau / consommation CPU.
- **Fix proposé** : ajouter `if(!axIsAdmin())return "Admin requis";` (comme `axInjectFunction`).

### BUG-6 — `_setApiKey` modal `_checkFirstSetup` autorise même non-admin
- **Sévérité** : MOYENNE
- **Lieu** : ligne 1597 `_checkFirstSetup` — guard `axIsAdmin()` présent au début, OK
- **Note** : déjà bloqué côté UI mais le bouton OK injecte `_setApiKey()` dans `onclick` HTML. Avec le guard ajouté en BUG-2, c'est désormais sûr.

## Bugs BAS (à voir)

### BUG-7 — Intervalles non nettoyés (memory leak léger)
- **Sévérité** : BASSE
- **Lieu** : 14+ `setInterval` au boot. La plupart sont stockés dans `_globalIntervals` mais certains comme ligne 4762 (`_axCancelExpiryInt`), 7846 (tour login), 956 (sentinels health), 7079 (SW update auto) ne sont jamais clear.
- **Impact** : sur PWA très long-running (>30j sans reload), légère consommation CPU.
- **Fix proposé** : ajouter `_globalIntervals.push(...)` à tous les setInterval, et un `_clearAllIntervals()` au logout.

### BUG-8 — `Notification.requestPermission` côté safari iOS sans test feature
- **Sévérité** : BASSE
- **Lieu** : ligne 495 `await Notification.requestPermission()` sans guard `typeof Notification!=="undefined"`
- **Impact** : iOS Safari (non PWA) lance une exception silencieuse.
- **Fix proposé** : ajouter `if(typeof Notification==="undefined")return false;` au début.

### BUG-9 — `localStorage.setItem` direct sur clés FB_FIX (3 cas restants)
- **Sévérité** : BASSE
- **Lieu** : ligne 5465 (snap.settings), 5461 (kb), 451 (ax_currency), etc.
- **Impact** : ces écritures ne se syncent PAS vers Firebase. Pas critique car déclenchées dans des contextes restore/migration où c'est intentionnel — mais à documenter.
- **Fix proposé** : commenter chaque `localStorage.setItem` direct sur clé FB_FIX avec `/* sync intentionnellement skip */`.

### BUG-10 — Plusieurs `await` dans des fonctions async OK mais fragile
- **Sévérité** : BASSE
- **Lieu** : `axScanBluetooth`, `axEncryptSecret`, `axNfcScan`, etc.
- **Note** : tout est correctement `async function` — pas de bug réel. À simplement vérifier avant nouveaux refactors.
