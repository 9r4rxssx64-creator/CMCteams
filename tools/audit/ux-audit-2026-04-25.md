### 2.2 Routes dupliquees dans `vMain()` switch

Avant :
```js
case"tokenbalance":return vTokenBalance();
case"servicespro":return vTokenBalance();
case"backupslist":return vBackupsList();   // vBackupsList(){return vBackups();}
```

Apres (tableaux de cases groupes) :
```js
case"tokenbalance":case"servicespro":return vTokenBalance();   /* alias canonique */
case"backupslist":return vBackups();                            /* alias canonique */
```

`vBackupsList` (stub 1 ligne) est conserve pour la compatibilite des liens externes mais n'est plus appele depuis le router (chemin court).

### 2.3 Icones identiques pour 2 concepts differents

| Concept | Avant | Apres |
|---------|-------|-------|
| `chat` (Chat IA) vs `messaging` (Messagerie admin) | ЁЯТм dans les deux | ЁЯТм chat / ЁЯУи messaging |

## 4. Standardisation des libelles

Verification effectuee :

- "Coffre" / "Coffre-fort" : 3 occurrences de "Coffre-fort" (tutorial guide, command palette, i18n map). Conserve : c'est la forme longue formelle, utilisee uniquement dans des contextes descriptifs. Tab labels et boutons restent "Coffre" partout.
- "Reglages" vs "Parametres" : "Parametres" n'apparait que dans des descriptions de liens externes (ex. "Parametres TVA Stripe", "Parametres projet Firebase") ou des `prompt()` natifs ("TV > Parametres > Reseau"). Aucune incoherence cote app.
- "KDMC" vs "APEX" : KDMC subsiste dans :
  - `ADMIN_ID="kdmc_admin"` (identifiant interne, NE PAS toucher тАФ clef Firebase de Kevin)
  - SVG apple-touch-icon (texte "KDMC" affiche en mosaique iOS) тАФ TODO Phase 2 (regenerer SVG)
  - URL Sentry `kdmc.sentry.io` et URL Vercel `kdmc-agent-monaco` (services externes тАФ non modifiables sans redeployer)
  - Quelques liens admin (Cloudflare worker `kdmc-proxy`) тАФ externe egalement

Decision : ne PAS renommer `kdmc_admin` ni les services externes. Le label visible "APEX" est deja partout cote UI. Seule l'icone PWA SVG merite refonte (a planifier separement).
