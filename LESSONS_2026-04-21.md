## 📚 15 leçons tirées cette session

### 1. Clé API multi-fallback cross-device (v12.31/v12.38)
**Problème** : Kevin voit "Clé API manquante" même avec clé déjà entrée ailleurs.
**Cause** : `_getApiKeyAsync` ne testait qu'un seul path Firebase.
**Fix** : 5 paths testés en cascade (`/apex/ax_shared_api_key`, `/apex/ax_api_key`, `/ax_shared_api_key`, `/shared/ax_api_key`, `/cmcteams/ax_shared_api_key`) + modal active avec input direct + diagnostic local visible.

### 2. Micro iPhone reste allumé quand app fermée (v12.50)
**Problème** : Sur iOS Safari PWA, `_sttRecog` (SpeechRecognition) ne se stoppe pas automatiquement au `pagehide`.
**Fix** : `axForceStopAllMedia()` + listeners `visibilitychange` + `pagehide` + `beforeunload` + `blur` + wrapper `getUserMedia` qui track tous les streams dans `window._activeMediaStreams`.
**Leçon** : toujours hook cleanup media sur les événements de cycle de vie PWA, pas juste `unload`.

### 3. Geoloc continue vs safety media (v12.51)
**Problème** : En coupant les streams media au `visibilitychange`, on coupait aussi le GPS.
**Fix** : Filter par `track.kind === "audio" || "video"` avant stop. GPS utilise geolocation API, pas MediaStream.
**Leçon** : séparer cleanup audio/vidéo de cleanup géoloc.

### 4. PR jamais mergée = déploiement fantôme (leçon #33 CLAUDE.md)
**Problème** : commits sur feature branch sans merge main → GitHub Pages ne voit rien.
**Fix** : **merger IMMÉDIATEMENT chaque batch stable**, pas à la fin.
**Règle** : à chaque commit stable (syntax OK + feature testable), PR + squash merge immédiat.

### 5. Autonomie expert vs minimum (règle permanente)
**Problème** : je livrais le minimum demandé, pas le maximum possible.
**Règle permanente CLAUDE.md** : si Kevin demande X, je fais X + Y (anticipation) + Z (amélioration). Jamais attendre validation.

### 6. Reproduire cross-app automatiquement (règle permanente)
**Problème** : je faisais Apex, Kevin devait redemander CMCteams.
**Règle** : chaque feature → check si applicable à l'autre app → si oui, appliquer sans demander.

### 7. Langage simple UI (v12.50)
**Problème** : "Erreur JS" / "HTTP 500" / "Timeout" incompréhensibles pour non-tech.
**Règle permanente** : table de correspondance 20 termes → français simple. Admin voit technique, clients jamais.

### 8. Jamais stocker seeds crypto / passwords banques (v12.44)
**Problème** : j'avais ajouté `ax_phantom_seed`, `ax_metamask_seed`, etc. dans Vault.
**Fix** : retiré 14 champs dangereux. AES-GCM PBKDF2 dispo pour le reste. Règle permanente : INTERDICTION seeds/passwords principaux.

### 9. Escalation autonome Apex → Claude Code (v12.49)
**Problème** : si IA + agents + sentinelles plantent, comment m'alerter quand Kevin est offline ?
**Fix** : Firebase `ax_claude_todo` + GitHub Action cron 2h + auto-issue + Telegram optionnel.

### 10. Multi-angles obligatoire IA (v12.41)
**Problème** : IA répondait 1 angle = manque de profondeur.
**Règle permanente system prompt** : direct + 2-3 alternatives + aller plus loin, toujours.

### 11. Trial 24h auto-assign (v12.47)
**Design** : nouveau user sans plan → auto-assign "trial 24h" → Kevin décide après.
**Bonus** : `axTrialStatus(uid)` pour savoir heures restantes.

### 12. Auto-handle erreurs avec escalade graduée (v12.48)
**Pattern** : erreur → auto-fix whitelist → si 3x/h → escalade client → si 5x/h → critical → Claude Code.

### 13. Reports journaliers par user (v12.52)
**Design** : `axTrackActivity` track tout → `axDailyReport` synthétise → admin interroge l'IA naturellement.

### 14. Détection actions suspectes (v12.52)
**Pattern** : whitelist actions admin-only + blacklist keywords suspects → bloque non-admin + alerte admin.

### 15. Update banner SW auto (v12.37+)
**Problème** : cache PWA iOS agressif, Kevin ne voyait pas les MAJ.
**Fix** : banner dorée automatique "🔄 Nouvelle version, tape pour recharger" dans les 2 apps.
