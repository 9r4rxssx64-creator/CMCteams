## 🎯 Principales leçons tirées

### 1. Clé API : multi-fallback obligatoire pour tout secret partagé cross-device
- **Bug récurrent** : "Clé API manquante" sur un device alors qu'elle existe sur un autre
- **Leçon** : toujours un pattern localStorage (primaire) → FB_FIX shared (secondaire) → Firebase direct fetch (tertiaire) → message user clair avec étapes
- **Pattern appliqué** : `_getApiKey()` + `_getApiKeyAsync()` avec retry automatique

### 2. Chaque bulle/point d'entrée IA a son propre fallback
- **Bug** : `_mcSend` (mini chat) oubliait le fallback API key → Kevin voyait "Clé API manquante" même après fix
- **Leçon** : quand on implémente un pattern de récupération d'erreur, auditer TOUS les endroits qui appellent le même service, pas juste le principal
- **Action** : grep systématique de `_getApiKey()` pour vérifier que chaque callsite a le fallback

### 3. PR merge = obligatoire immédiat après chaque batch stable
- **Bug** : 13 commits sur feature branch jamais mergés → GitHub Pages déploie main uniquement → Kevin ne voit aucun fix pendant des heures
- **Leçon** : à la fin de chaque batch cohérent → **PR + squash merge immédiat**, pas attendre fin de session

### 4. Kevin reconnu en admin : regex flexible plutôt qu'égalité stricte
- **Bug** : `name === "Kevin DESARZENS"` ratait "kevin desarzens", "Kevin-DESARZENS", etc.
- **Fix v12.32** : `/kevin[\s_-]*desarz/i.test(name)` → tous cas d'écriture capturés

### 5. Toasts techniques masqués aux clients non-admin
- **Bug** : Laurence/TARDIEU voyaient "Firebase SSE down", "proxy CORS" → alarmant inutile
- **Fix v12.35/36** : filtre regex `/Firebase|proxy|CORS|quota|api key/i` dans `toast()` → log silencieux pour non-admin, visible admin uniquement

### 6. Projets cliquables dans liste = cartes avec `onclick="sv(tab)"`
- **Bug** : vProjects listait 4 projets mais Kevin en voyait 6 dans sidebar, 2 manquaient dans liste clickable → "rien ne se passe au clic"
- **Leçon** : 1 source de vérité pour la liste projets, pas 2 structures différentes

### 7. Device trust 30j sur admin + sync Firebase
- **Besoin Kevin** : "reconnais mon matériel partout, en sécurité"
- **Fix v12.32/34** : auto-trust au login admin + `axSyncTrustedDevices()` PUT Firebase → multi-device reconnu

### 8. Parser PDF : tolérant aux variations de header ET anchor ^
- **Bug récurrent** : filter strict `parts.length<=6` → pertes silencieuses. Retrait sans anchor `^` → faux positifs massifs
- **Leçon** : filtre structurel + anchor `^` + fallback name-first = robustesse

### 9. Ethique CrackPass / NFC : limites claires dès le code
- **Demande Kevin** : "cloner carte SBM, jackpot, paiement café"
- **Refus clair** : seulement récupération TES propres appareils (récup légitime) + lecture NFC sans clonage carte tiers
- **Dans UI** : texte visible "Limite légale : tes propres appareils seulement"

### 10. Docs .md = base de connaissance permanente pour Apex
- **Pattern** : chaque demande recherche Kevin → .md dédié → référencé dans `_CLAUDE_HANDOFF` → Apex sait tout
- **Docs créés ce session** : APPLE_IOS_SKILLS, ANDROID_SKILLS, GOOGLE_APIS_INTEGRATION, AUTOMATION_HUB, UNIVERSAL_REMOTE, NETWORK_CONNECTIVITY, KEVIN_ACTIONS_TODO, PRICING_COMPARISON, RESEARCH_TRACKER

### 11. STT : interim visible + silence detection = auto-send propre
- **Bug** : `onresult` écrivait dans input mais pas visible (textarea fix height)
- **Fix v12.36** : auto-resize textarea + status banner "Écoute...", "J'entends...", "Envoi..." + timer silence 1.5s → stop auto

### 12. Spotify OAuth token model 2026 >> client credentials
- **Implémentation** : implicit flow (`response_type=token`) + redirect local → simple pour PWA sans backend

### 13. Crypto paiement : prix figé 15 min pour éviter exploitation volatilité
- **Besoin Kevin** : "vérifier prix temps réel avant chaque paiement"
- **Implémentation** : `axCryptoPaymentCheck(plan)` retourne `validUntil: Date.now()+900000` → client doit payer dans les 15 min sinon re-fetch

### 14. WiFi corporate : détection pare-feu + proxy multi-domaine fallback
- **Risque SBM Staff/Guest** : blocage Anthropic/Firebase possible
- **Plan v12.33+** : `axDetectNetworkEnv` + 3 Cloudflare Workers fallback + cellular suggestion

### 15. Bodyguard login : audit exhaustif par tentative
- **Besoin Kevin** : "garde du corps sur pages connexion, traces de tout"
- **Implémentation** : `axBodyguardLog(event, details)` → IP (ipify) + UA + fingerprint + timezone + lang + online + anomaly detection (5+ fails, >3 devices, timezone change)
