# i18n — Internationalisation Apex AI + CMCteams

Version : Apex v12.254 / CMCteams v9.534 (2026-04-25)

## Objectif

Infrastructure i18n minimale (30 strings critiques) pour Apex AI + CMCteams. Permet de basculer FR/EN/IT sans refactor massif. Fallback automatique sur FR si la cle/langue manque.

## Architecture

### Apex AI (apex-ai/index.html)
- Dictionnaire global `var I18N={fr:{},en:{},it:{}}`
- Helper `axT(key)` — retourne `I18N[K.settings.lang||'fr'][key] || I18N.fr[key] || key`
- Lang stockee dans `K.settings.lang` (selectable dans Reglages)

### CMCteams (index.html)
- Dictionnaire global `var I18N={fr:{},en:{},it:{}}` (meme structure)
- Helper `cmcT(key)` — retourne `I18N[A.user.lang||'fr'][key]`
- Lang stockee dans `A.user.lang` ou `lg("cmc_lang","fr")`

## Ajouter une langue

1. Dans `apex-ai/index.html` (et/ou `index.html`), trouver `var I18N={`
2. Copier le bloc `fr:` en entier
3. Renommer la cle (ex: `de:` pour allemand, `es:` pour espagnol)
4. Traduire chaque valeur
5. Bumper APP_VER + commit

Exemple :
```js
de:{
  "login.required":"Bitte zuerst anmelden",
  "login.welcome":"Willkommen!",
  // ... etc
}
```

## Utilisation dans le code

Au lieu de :
```js
toast("Connecte-toi d'abord","warn");
```

Ecrire :
```js
toast(axT("login.required"),"warn"); // Apex
toast(cmcT("login.required"),"warn"); // CMCteams
```

## SEO multilingue

Hreflang ajoutes dans `apex-ai/index.html` :
- `fr` (defaut, sans param)
- `en` (`?lang=en`)
- `it` (`?lang=it`)
- `de` (`?lang=de`)
- `es` (`?lang=es`)
- `x-default` (canonique)

Note : ces URLs sont des indices SEO. Le code lit `K.settings.lang` en priorite, mais peut aussi parser `?lang=` au boot pour matcher avec l'index Google.

## Fallback comportement

```
axT("missing.key") -> "missing.key" (literal returned, no error)
axT("login.required") avec lang="zz" -> retourne FR fallback
axT("login.required") avec lang="en" + cle absente en EN -> retourne FR fallback
```

Fail-safe : aucune exception levee, l'app ne casse jamais sur i18n.
