# 📒 Commandes (slash-commands) — liste complète

Tape `/` dans Claude Code pour les voir et les lancer. Chaque commande applique un « mode » de réflexion à ce que tu écris après.

> Mise à jour temps réel : toute nouvelle commande ajoutée dans ce dossier apparaît automatiquement ici.

## 🧠 Analyse & réflexion
| Commande | Définition |
|---|---|
| `/analyst` | Analyse approfondie et structurée du sujet donné |
| `/deepdive` | Exploration très détaillée et exhaustive du sujet |
| `/firstprinciples` | Raisonnement depuis les principes fondamentaux |
| `/compare` | Compare plusieurs options de façon structurée |
| `/matrix` | 🆕 Présente les options et compromis dans une matrice claire |
| `/proscons` | Liste les avantages et les inconvénients |

## 🔍 Critique & challenge
| Commande | Définition |
|---|---|
| `/critic` | Trouve uniquement les failles, défauts et risques |
| `/contrarian` | Challenge l'idée — joue l'avocat du diable |
| `/challenge` | 🆕 Avocat du diable — attaque les hypothèses, stress-test l'idée |
| `/roast` | 🆕 Feedback brutalement honnête, sans filtre (cash mais utile) |

## ✍️ Écriture & clarté
| Commande | Définition |
|---|---|
| `/simplify` | Explique simplement, comme à un débutant |
| `/eli5` | Explication ultra-simple (comme à un enfant de 5 ans) |
| `/brief` | 🆕 Condense à l'essentiel (TL;DR ultra-court et actionnable) |
| `/focus` | 🆕 Coupe le bruit, ne garde que ce qui compte vraiment |
| `/ghost` | 🆕 Réécrit pour sonner 100% humain (enlève les tics d'IA) |

## 🚀 Aller plus loin
| Commande | Définition |
|---|---|
| `/optimizer` | Améliore et optimise ce qui est fourni |
| `/expand` | 🆕 Explore les implications, cas d'usage et extensions |
| `/steal` | 🆕 Reverse-engineer ce qui marche (et comment le reproduire) |
| `/systemize` | 🆕 Transforme un coup unique en système / workflow réutilisable |

---

🆕 = ajouté le 2026-06-19 (les « 10 Claude Commands »).

**Comment ajouter une commande perso :** crée un fichier `.claude/commands/<nom>.md` avec un en-tête `description:` + `argument-hint:`, puis le texte du mode. Elle apparaît aussitôt dans `/` et dans cette liste.
